# 寄存器

> [!info] 这篇笔记是干嘛的
> 用最白话的方式把"寄存器"这个概念讲明白。不堆术语，上来就理解。

相关笔记：[[GPIO和AFIO]] [[32学习笔记]]

---

## 用生活例子理解寄存器

想象你住进了一家酒店，房间里有一排开关面板：

```
┌──────────────────────────────┐
│  房间控制面板                  │
│                              │
│  [1] 灯      [2] 空调         │
│  [3] 窗帘    [4] 电视         │
│  [5] 门锁    [6] 插座         │
│                              │
│  你拨哪个开关 → 哪个设备就动    │
└──────────────────────────────┘
```

STM32 芯片内部也是这样。每个引脚、每个外设都连着一组"开关面板"，这就是**寄存器**。区别在于：

- 酒店里你用手拨开关 → STM32 里 CPU 用数字（0 和 1）来"拨"
- 酒店里每个开关有编号 → STM32 里每个"开关"有一个**地址**
- 你拨开关灯就亮 → CPU 往地址写 1，对应的引脚就输出高电平

> [!tip] 记住这句话就够了
> **寄存器 = 芯片内部一排有地址的开关，CPU 往地址写 0 或 1，硬件就跟着动作。**

---

## CPU 是怎么找到寄存器的

CPU 操作寄存器，就跟你在程序里读写一个变量一样。唯一的区别是：普通变量的地址在 RAM 里，寄存器的地址在另一个区域。

STM32 把所有东西放在同一个地址空间里：

| 你操作的东西 | 在哪个地址区域 | 举例 |
|:-------------|:---------------|:-----|
| 你写的程序代码 | `0x0800_0000` 起（Flash） | 你写的 main 函数 |
| 你的变量、数组 | `0x2000_0000` 起（SRAM） | `int a = 0;` |
| **外设的寄存器** | `0x4000_0000` 起 | GPIO、USART、TIM 的寄存器 |
| 内核的寄存器 | `0xE000_0000` 起 | NVIC、SysTick |

对 CPU 来说，读变量和读寄存器**没有区别**，都是"去某个地址拿数据"。

---

## 寄存器地址怎么算

每个外设的寄存器地址 = **基地址 + 偏移**

打个比方：酒店在"建设路 100 号"（基地址），大堂在 1 楼（偏移 +0），餐厅在 2 楼（偏移 +1），健身房在 3 楼（偏移 +2）。

STM32 里：

```
GPIOC 的"门牌号"（基地址）= 0x4001_1000

GPIOC 里面的"房间"（寄存器）：
  基地址 + 0x00 = 0x4001_1000 → CRL   （配 Pin0~7）
  基地址 + 0x04 = 0x4001_1004 → CRH   （配 Pin8~15）
  基地址 + 0x08 = 0x4001_1008 → IDR   （读引脚电平）
  基地址 + 0x0C = 0x4001_100C → ODR   （设输出电平）
  基地址 + 0x10 = 0x4001_1010 → BSRR  （安全地改某一位）
  基地址 + 0x14 = 0x4001_1014 → BRR   （安全地清某一位）
```

> [!note] 不用背
> 翻开参考手册，每个外设章节开头都有一张表，列好了所有寄存器的偏移地址。用的时候查就行。

常用外设基地址：

| 外设 | 基地址 | 挂在哪条总线 |
|:-----|:-------|:-------------|
| GPIOA | `0x4001_0800` | APB2 |
| GPIOB | `0x4001_0C00` | APB2 |
| GPIOC | `0x4001_1000` | APB2 |
| AFIO  | `0x4001_0000` | APB2 |
| RCC   | `0x4002_1000` | APB2 |
| USART1| `0x4001_3800` | APB2 |
| TIM2  | `0x4000_0000` | APB1 |

## 位操作：怎么"拨"寄存器里的某一位

寄存器是 32 位的，你经常只想改其中一位，不想影响其他位。

> [!warning] 新手最容易犯的错
> 直接写 `寄存器 = 1`，这会把其他 31 位全部变成 0！如果那些位还控制着别的引脚，别的引脚就跟着乱了。

正确的做法是用位操作：

```c
// 只把第 n 位变成 1，其他位不动
reg |= (1 << n);

// 只把第 n 位变成 0，其他位不动
reg &= ~(1 << n);

// 看看第 n 位是 0 还是 1
val = (reg >> n) & 1;
```

用 `|=` 的意思是"在原有基础上再加"，用 `&= ~` 的意思是"只把这一位拿掉"。就像你只想开灯不想动空调，就不能把整个面板全按一遍。

## GPIO 的 7 个寄存器

GPIO 是你最先接触的外设，它的 7 个寄存器就是理解所有外设的"模板"。

| 寄存器 | 干什么用的 | 读写 |
|:------|:----------|:----:|
| CRL   | 配 Pin0~7 是输入还是输出、什么模式 | 可读可写 |
| CRH   | 配 Pin8~15 是输入还是输出、什么模式 | 可读可写 |
| IDR   | 读引脚现在的实际电平是高还是低 | 只读 |
| ODR   | 设引脚输出高还是低 | 可读可写 |
| BSRR  | 安全地把某一位置 1 或清 0 | 只写 |
| BRR   | 安全地把某一位清 0 | 只写 |
| LCKR  | 锁住配置不让改 | 可读可写 |

### CRL / CRH：每个引脚占 4 个 bit

CRL 管 Pin0~7，CRH 管 Pin8~15。每个引脚占 4 个 bit，拆成两半：

```
  高 2 位          低 2 位
┌──────────┬──────────┐
│   CNF    │   MODE   │
│ 选什么功能 │ 选输入还是输出│
└──────────┴──────────┘
```

**MODE（先决定大方向）**：
- `00` = 输入（上电默认就是这个）
- `01` = 输出，最快 10MHz
- `10` = 输出，最快 2MHz
- `11` = 输出，最快 50MHz

**CNF（再决定具体类型）**：

如果 MODE 选了输入（`00`）：
- `00` = 模拟输入（给 ADC 用的）
- `01` = 浮空输入（引脚悬着，电平不确定）
- `10` = 上拉/下拉输入（默认给个高/低电平）
- `11` = 保留（不用）

如果 MODE 选了输出（`01/10/11`）：
- `00` = 推挽输出（最常用，直接驱动 LED）
- `01` = 开漏输出（I2C 等通信用）
- `10` = 复用推挽输出（交给片上外设控制）
- `11` = 复用开漏输出

> [!example] 算一遍：PC13 配成推挽输出 50MHz
> 1. PC13 ≥ 8，所以用 CRH
> 2. Pin13 在 CRH 里是第 4 组 4bit，即 bit[23:20]
> 3. 推挽输出 50MHz → CNF=`00`, MODE=`11` → 4bit = `0011` = `0x3`
> 4. 操作：
>    ```c
>    // 先清零这 4 位
>    *(volatile unsigned int *)0x40011004 &= ~(0xF << 20);
>    // 再写入 0x3
>    *(volatile unsigned int *)0x40011004 |=  (0x3 << 20);
>    ```

### BSRR 和 ODR 的区别

两个都能控制输出电平，但用法不同：

- **ODR**：你改一位之前要先读出整个 32 位的值，改完再写回去。如果在读和写之间被中断打断，就可能出问题。
- **BSRR**：你直接写就行，写 1 的位会被置位，写 0 的位不受影响。==一步到位，不会被打断==。

> [!tip] 简单记
> 改一个引脚的电平，优先用 BSRR/BRR，不要直接写 ODR。

### IDR：随时能读

不管引脚配成了输入还是输出，IDR 都能读到引脚的**真实电平**。

## 最重要的一步：开时钟

> [!danger] 90% 的新手都会踩的坑
> 寄存器全配对了，但引脚就是没反应。原因：**忘了开时钟**。

STM32 为了省电，所有外设的时钟默认是关着的。你得先在 RCC（复位和时钟控制）寄存器里把对应外设的时钟打开，后面的配置才有效。

RCC 里有个寄存器叫 `RCC_APB2ENR`（APB2 外设时钟使能），地址 `0x4002_1018`：

| 位 | 控制谁 |
|:--:|:-------|
| bit0 | AFIO 的时钟 |
| bit2 | GPIOA 的时钟 |
| bit3 | GPIOB 的时钟 |
| bit4 | GPIOC 的时钟 |
| bit11 | USART1 的时钟 |

```c
// 打开 GPIOC 的时钟（bit4 写 1）
*(volatile unsigned int *)0x40021018 |= (1 << 4);
```

## 三种写代码的方式

操作同一个寄存器，有三种写法，从底层到上层：

### 1. 裸地址（帮你理解原理）

```c
// 开 GPIOC 时钟
*(volatile unsigned int *)0x40021018 |= (1 << 4);

// 配 PC13 推挽输出 50MHz
*(volatile unsigned int *)0x40011004 &= ~(0xF << 20);
*(volatile unsigned int *)0x40011004 |=  (0x3 << 20);

// PC13 输出高电平
*(volatile unsigned int *)0x40011010 = (1 << 13);
```

> [!question] 为什么要加 volatile
> 编译器看到你对同一个地址写了值，可能会"自作聪明"地优化掉后续的读取。但寄存器的值会被硬件随时改变（比如引脚电平变了），加了 `volatile` 就是告诉编译器："这个地址的内容随时会变，每次都要重新读，别偷懒。"

### 2. 结构体映射（标准库的底层）

裸地址太难记。标准库用一个结构体把寄存器包起来：

```c
typedef struct {
    volatile uint32_t CRL;
    volatile uint32_t CRH;
    volatile uint32_t IDR;
    volatile uint32_t ODR;
    volatile uint32_t BSRR;
    volatile uint32_t BRR;
    volatile uint32_t LCKR;
} GPIO_TypeDef;

#define GPIOC ((GPIO_TypeDef *)0x40011000)

// 用起来就很自然了
GPIOC->CRH &= ~(0xF << 20);
GPIOC->CRH |=  (0x3 << 20);
GPIOC->BSRR = (1 << 13);
```

`GPIOC->CRH` 编译后就是访问 `0x40011004`，和裸地址完全一样。

### 3. 库函数（实际开发最常用）

```c
#include "stm32f10x.h"

// 开时钟
RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOC, ENABLE);

// 配引脚
GPIO_InitTypeDef GPIO_InitStruct;
GPIO_InitStruct.GPIO_Pin   = GPIO_Pin_13;
GPIO_InitStruct.GPIO_Mode  = GPIO_Mode_Out_PP;
GPIO_InitStruct.GPIO_Speed = GPIO_Speed_50MHz;
GPIO_Init(GPIOC, &GPIO_InitStruct);

// 输出高/低电平
GPIO_SetBits(GPIOC, GPIO_Pin_13);
GPIO_ResetBits(GPIOC, GPIO_Pin_13);
```

> [!abstract] 三种方式的关系
> 库函数封装了结构体，结构体映射了裸地址。不管用哪种，CPU 最终做的事都一样：**往某个地址写值**。理解了寄存器，用库函数时心里才有底。

库函数和寄存器的对应关系：

| 库函数 | 底层在写哪个寄存器 |
|:------|:------------------|
| `RCC_APB2PeriphClockCmd()` | RCC_APB2ENR |
| `GPIO_Init()` | CRL / CRH |
| `GPIO_SetBits()` | BSRR |
| `GPIO_ResetBits()` | BRR |
| `GPIO_ReadInputDataBit()` | IDR |
| `USART_SendData()` | DR |
| `USART_GetFlagStatus()` | SR |

## 其他外设的寄存器都一个套路

学完 GPIO 后你会发现，其他外设（USART、TIM、ADC……）的寄存器都可以分成这几类：

| 类型 | 干什么 | 例子 |
|:-----|:-------|:-----|
| 配置寄存器 | 设工作模式和参数 | GPIO 的 CRL/CRH |
| 数据寄存器 | 读/写实际数据 | GPIO 的 IDR/ODR、USART 的 DR |
| 状态寄存器 | 查外设现在什么状态 | USART 的 SR（发完了？收到了？） |
| 时钟寄存器 | 开关外设 | RCC 的各种 ENR |

> [!tip] 遇到新外设别慌
> 套路都是：**开时钟 → 配模式 → 读写数据 → 查状态**。把 GPIO 这套流程吃透，换一个外设就是换一章手册的事。

## 踩坑记录

| 现象 | 原因 | 怎么解决 |
|:-----|:-----|:---------|
| 配完了没反应 | 忘了开时钟 | 先 `RCC_APBxPeriphClockCmd()` |
| 电平不对 | CRL/CRH 的 4bit 写错 | 对照手册 CNF/MODE 表仔细算 |
| 改一个引脚别的引脚也变了 | 直接写了 ODR | 改用 BSRR/BRR |
| 写了寄存器没效果 | 没加 volatile | 指针必须加 `volatile` |
