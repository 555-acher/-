# 嵌入式传感器入门（附原理图对比）

> [!info] 这篇笔记解决什么问题
> 刚接触嵌入式的人面对一堆传感器模块不知道怎么选、怎么接、怎么写代码。这篇笔记按类型对比讲解常用传感器，配上清晰的原理图和接线方式，帮你快速上手。

相关笔记：[[寄存器]] [[GPIO和AFIO]] [[32DAY1]]

---

## 一、传感器是干嘛的

单片机只能读懂**电信号**（高低电平、电压数值），但现实世界的温度、距离、光线、湿度、震动都是物理量。

**传感器就是翻译官**：把物理量 → 转换成单片机能读懂的电信号。

```
  现实世界                    传感器                    单片机

  温度 25℃ ──────┐                               ┌──────→ PA0 读取
  距离 30cm ──────┤    ┌─────────────┐           │
  光线 强   ──────┼───→│   传感器模块  │──电信号───┼──────→ PA1 ADC采集
  湿度 60%  ──────┤    │  (转换器)     │           │
  震动 有   ──────┘    └─────────────┘           └──────→ PA2 中断检测
```

### 嵌入式最常用的 4 大类

| 类型 | 输出信号 | 特点 | 代表传感器 |
|:-----|:---------|:-----|:-----------|
| 数字输出 | I2C / 单总线 / 串口 | 数据稳定，直接读数字 | DHT11、MPU6050、TSL2561 |
| 模拟输出 | 连续变化的电压 | 需要 ADC 读取 | 光敏电阻、热敏电阻、MQ-2 |
| 开关型 | 只有高/低电平 | 最简单，测有无 | 红外避障、霍尔开关、限位开关 |
| 脉冲型 | 脉冲信号 | 靠计数测速度 | 霍尔编码器、红外测速 |

> [!tip] 新手首选
> 刚入门选**数字输出型**，因为不需要处理模拟信号，直接读数字，最省心。

## 二、分类型讲解 + 原理图对比

### 1. 单总线温湿度传感器：DHT11 VS DHT22

#### 核心区别

| 参数 | DHT11 | DHT22 (AM2302) |
|:-----|:------|:---------------|
| 温度精度 | ±2℃ | ±0.5℃ |
| 湿度精度 | ±5% | ±2% |
| 温度量程 | 0~50℃ | -40~80℃ |
| 价格 | 便宜（~3元） | 贵（~15元） |
| 适用场景 | 教学、实验 | 工业采集、长期监测 |

#### 通用引脚定义（两者完全一致）

| 引脚 | 名称 | 说明 |
|:----:|:-----|:-----|
| 1 | VCC | 供电 3.3V~5V |
| 2 | DATA | 单总线数据引脚 |
| 3 | NC | 空脚，不接 |
| 4 | GND | 电源地 |

#### 原理图

```
  3.3V ────────────────┬──────────────────────┐
                       │                      │
                    ┌──┴──┐                   │
                    │10KΩ │  上拉电阻          │
                    │     │                   │
                    └──┬──┘                   │
                       │                      │
                       ├────── DATA(引脚2)    │
                       │                      │
  STM32                │               ┌──────┴──────┐
  PA0 ─────────────────┘               │   DHT11/22  │
                                       │             │
  3.3V ────────────────────────────────│ VCC    GND  │─── GND
                                       └─────────────┘
```

> [!warning] 必须加上拉电阻
> DATA 线必须接 10KΩ 上拉电阻到 VCC。因为单总线在空闲时需要保持高电平，没有上拉电阻信号会不稳定，读不到数据。

> [!tip] 供电注意
> 推荐用 3.3V 供电。虽然 DHT11 标称支持 5V，但长期 5V 容易缩短寿命，而且 5V 信号直接接 STM32 的 3.3V 引脚有风险。

#### 代码要点

```c
// 1. 开 GPIO 时钟
RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOA, ENABLE);

// 2. PA0 配置为开漏输出（单总线通信需要双向切换）
//    主机发送请求时：输出模式
//    接收传感器响应时：输入模式
// 3. 按照时序协议读取 40bit 数据（8bit湿度 + 8bit湿度校验 + 8bit温度 + 8bit温度校验 + 8bit校验和）
```

### 2. I2C 姿态传感器：MPU6050 VS MPU9250

#### 核心区别

| 参数 | MPU6050 | MPU9250 |
|:-----|:--------|:--------|
| 轴数 | 6轴（加速度+陀螺仪） | 9轴（加速度+陀螺仪+磁力计） |
| 能测什么 | 倾斜、震动、加速度 | 倾斜、震动、加速度、方向 |
| 典型应用 | 平衡小车、四轴飞行器 | 电子罗盘、户外导航、平衡小车 |
| I2C 地址 | 0x68 | 0x68（加速度陀螺仪）/ 0x0C（磁力计） |

#### 原理图（两款硬件电路完全通用）

```
  3.3V ────────────────┬──────────────────────┬──────────────────┐
                       │                      │                  │
                    ┌──┴──┐                ┌──┴──┐                │
                    │4.7KΩ│  SDA上拉        │4.7KΩ│  SCL上拉       │
                    └──┬──┘                └──┬──┘                │
                       │                      │                  │
  STM32                │                      │           ┌──────┴──────┐
  PB7 (I2C1_SDA) ──────┘                      │           │  MPU6050 /  │
  PB6 (I2C1_SCL) ─────────────────────────────┘           │  MPU9250    │
                                                           │             │
  3.3V ──────────────────────────────────────────────────│ VCC         │
  GND ────────────────────────────────────────────────────│ GND         │
                                                           │ SDA   SCL   │
                                                           └─────────────┘
```

> [!note] 为什么 I2C 要上拉
> I2C 总线是"开漏"结构，引脚只能拉低，不能自己拉高。上拉电阻负责在总线空闲时把电平拉高。没有上拉电阻 = I2C 完全不工作。

> [!warning] 用前必须开 I2C 时钟
> ```c
> RCC_APB1PeriphClockCmd(RCC_APB1Periph_I2C1, ENABLE);   // I2C1 外设时钟
> RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOB, ENABLE);   // GPIOB 时钟
> ```
> 忘了开时钟，硬件电路再正确也读不到任何数据。

### 3. 光敏传感器：模拟光敏模块 VS TSL2561

#### 模拟光敏电阻模块

```
                    光照越强 → 电阻越小 → 分压点电压越低

   3.3V ─────┬─────────────┐
             │             │
          光敏电阻 RG     │  ← 受光照影响，阻值变化
             │             │
             ├─────────────┼──── OUT ──── PA1 (ADC1通道1)
             │             │
          1KΩ 固定电阻     │  ← 分压电阻
             │             │
   GND ─────┴─────────────┘

   ADC 读取公式：电压 = ADC值 / 4096 × 3.3V
   光越强 → 电压越低 → ADC 值越小
```

> [!example] 原理
> 光敏电阻和固定电阻串联分压。光照越强，光敏电阻阻值越小，分压点电压越低。单片机通过 ADC 读取这个电压值，就能间接知道光照强度。
>
> 缺点：精度差、受温度影响大、只能比较相对亮暗，不能测出精确的光照值（lux）。

#### TSL2561 数字光照传感器（I2C）

```
   电路和 MPU6050 完全一样（I2C + 上拉电阻）：

   3.3V ────────┬───────────┬──────┐
                │           │      │
             4.7KΩ       4.7KΩ     │
                │           │   ┌──┴──────┐
                │           │   │ TSL2561 │
   PB7(SDA) ────┘           │   │         │
   PB6(SCL) ────────────────┘   │ VCC GND │── GND
                                │ SDA SCL │
                                └─────────┘

   优点：直接输出光照强度数值（lux），精度高，不受温度干扰
   适用：智能家居、相机曝光、户外光照监测
```

#### 两者对比

| 对比项 | 光敏电阻模块 | TSL2561 |
|:-------|:------------|:--------|
| 信号类型 | 模拟（ADC 读取） | 数字（I2C 读取） |
| 电路复杂度 | 简单，分压电路即可 | 需要上拉电阻 |
| 精度 | 低，只能比较相对亮暗 | 高，直接输出 lux 值 |
| 价格 | 极便宜（~1元） | 贵（~10元） |
| 适用场景 | 课堂实验、简易光控灯 | 户外光照检测、智能照明 |

### 4. 红外传感器：避障 VS 对射

#### 红外避障模块（反射式）

```
         ┌──────────────────────┐
         │    红外避障模块        │
         │                      │
         │  ┌───┐      ┌───┐   │
         │  │发射│ ───→ │接收│   │  ← 前方有障碍物时，红外光被反射回来
         │  │ LED│  反射 │管  │   │
         │  └───┘       └───┘   │
         │     ↑                   │
         │   有障碍物              │
         └──┬───┬───┬─────────────┘
            │   │   │
           VCC OUT GND
            │   │   │
            │   │   └─── GND
            │   │
   3.3V ────┘   └──────────── PA2 (GPIO 输入)

   逻辑：
     前方有障碍 → 红外反射回来 → OUT = 低电平（0）
     前方无障碍 → 没有反射     → OUT = 高电平（1）

   模块上通常有电位器可调节检测距离（2~30cm）
```

#### 红外对射传感器（遮挡式）

```
   发射端                        接收端
   ┌───────┐                    ┌───────┐
   │ 红外   │═══════════════════│ 接收管 │
   │ 发射管 │     红外光路       │       │
   │       │  ─────────────→   │       │
   └───┬───┘                    └───┬───┘
       │                            │
   3.3V──┬───┐                3.3V──┬───┐
         │220Ω│                     │10KΩ│  上拉
         │   │                      │   │
   GND ──┴───┘                 GND ─┴───┘
                                     │
                                    OUT ──── PA3 (GPIO 输入)

   逻辑：
     光路无遮挡 → 接收管导通 → OUT = 低电平（0）
     光路被遮挡 → 接收管截止 → OUT = 高电平（1）

   适用：流水线计数、闸门检测、限位开关
```

## 三、通用硬件规则（小白必背）

> [!danger] 违反任何一条都可能导致传感器不工作

### 1. 供电要匹配

| 传感器供电 | 接 STM32 的 | 注意事项 |
|:----------|:-----------|:---------|
| 3.3V | 3.3V 引脚 | 直接接，没问题 |
| 5V | 不能直接接 3.3V | 需要 5V 电源单独供电，信号线要电平转换 |
| 3.3V~5V 宽压 | 优先接 3.3V | 信号电平和 STM32 匹配，最安全 |

### 2. 必须共地

单片机的 GND 和传感器的 GND **必须连在一起**。不共地 = 没有参考点 = 信号乱跳。

```
   ┌─────────┐                    ┌──────────┐
   │ STM32   │                    │  传感器   │
   │         │    信号线          │          │
   │  PA0 ───┼────────────────────┼── DATA   │
   │         │                    │          │
   │  GND ───┼────────────────────┼── GND    │  ← 必须连在一起！
   └─────────┘                    └──────────┘
```

### 3. 通讯总线必须加上拉

| 总线类型 | 上拉电阻 | 接法 |
|:---------|:---------|:-----|
| 单总线（DHT11/22） | 10KΩ | DATA 线上拉到 VCC |
| I2C（MPU6050、TSL2561） | 4.7KΩ | SDA 和 SCL 各上一个 |

> [!warning] 不加上拉会怎样
> 数据偶尔丢失、读取失败、读到全 0 或全 1。有时候能读到有时候读不到，这种"时好时坏"的问题 90% 是上拉电阻没接。

### 4. 模拟输入不能接 5V

STM32 的 ADC 输入范围是 0~3.3V。如果传感器输出 5V 信号直接接到 ADC 引脚，会烧坏 IO 口。

### 5. 用前必须开 RCC 时钟

> [!danger] 新手第一大坑
> 硬件电路完全正确，但读不到任何数据 —— 99% 是忘了开时钟。

| 传感器类型 | 接的接口 | 要开哪些时钟 |
|:----------|:---------|:------------|
| DHT11（单总线） | GPIO | GPIOA/B/C 时钟 |
| MPU6050（I2C） | I2C1 | I2C1 外设时钟 + GPIOB 时钟 |
| 光敏电阻（ADC） | ADC1 | ADC1 时钟 + GPIOA 时钟 |
| 红外避障（开关） | GPIO | GPIOA/B/C 时钟 |

```c
// 示例：用 MPU6050 前必须开两个时钟
RCC_APB1PeriphClockCmd(RCC_APB1Periph_I2C1, ENABLE);   // I2C1
RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOB, ENABLE);   // GPIOB（SDA/SCL 在 PB6/PB7）
```

## 四、快速选型口诀

| 场景 | 选什么 | 理由 |
|:-----|:-------|:-----|
| 只测温湿度、低成本实验 | DHT11 | 便宜够用 |
| 需要精准温湿度长期采集 | DHT22 | 精度高、量程大 |
| 小车平衡、测倾斜角度 | MPU6050 | 6轴够用 |
| 需要指南针、户外姿态 | MPU9250 | 9轴带磁力计 |
| 简单光控、入门练习 | 光敏电阻模块 | 最便宜最简单 |
| 精准光照数值采集 | TSL2561 | 直接出 lux 值 |
| 小车避障检测 | 红外反射避障模块 | 简单可靠 |
| 流水线计数、闸门检测 | 红外对射传感器 | 精确触发 |

```

## 六、总结

> [!abstract] 一句话记住
